👤 ユーザー層

  • 予約者：一般客（PC・スマホ）
  • スタッフ：上村さん他（タブレット・PC）

🌐 フロントエンド GitHub Pages

  • index.html — 予約者向け予約UI
  • staff.html — スタッフ詳細管理（30秒自動更新）
  • staff2.html — 月俯瞰ビュー（カテゴリフィルタ）

↓ REST API（CORS制御）

⚙️ バックエンド API Cloud Functions 2nd gen / Node.js 20 / TypeScript

  • GET /availability — 占有スロット一覧（公開）
  • POST /createReservation — 予約作成（公開・トランザクション）
  • GET /listReservations — 予約一覧（要APIキー）
  • PATCH /updateReservation — ステータス更新
  • DELETE /cancelReservation — 予約キャンセル
  • GET/POST /futamiDays — ふたみの日リスト
  • GET /health — ヘルスチェック
↓ Firestore SDK

💾 データベース Firestore (asia-northeast1)

  • reservations/{id} — 予約本体（顧客情報・期間・状態）
  • slots/{key} — 通常スロット roomId|YYYY-MM-DD|hour
  • shared_slots/{key} — ふたみの日サウナ（capacity:8 / used:N）
  • tennis_slots/{key} — テニス30分単位 court_X|YYYY-MM-DD|HHMM
  • config/special_days — ふたみの日リスト24ヶ月分

📋 特殊ルール

🌅 ふたみの日（毎月23日）

  • 料金：1名2,300円（税込）
  • 定員：1枠最大8名（複数組同時予約OK）
  • 対象時間枠：A 10-12 / B 12:30-14:30 / C 15-17 / D 17:30-19:30
  • 振替ルール：23日が火曜なら前日（22日）に振替（火曜定休のため）
  • 管理：Firestoreの config/special_days で柔軟に変更可能

🎾 テニス

  • 30分刻み開始：9:00, 9:30, 10:00, 10:30...
  • 1時間単位の予約：1予約 = 30分×2スロット
  • 重複防止：自分の選択枠と重なる時間は自動でグレーアウト
  • 火曜定休：管理画面で「定休日」と表示

🧖 サウナ通常

  • 4枠制：A 10-12 / B 12:30-14:30 / C 15-17 / D 17:30-19:30
  • 料金：12,000円/グループ（貸切型）
  • 23日のみ：ふたみの日プランに自動切替
